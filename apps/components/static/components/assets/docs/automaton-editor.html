Representation

The property automaton accepts two formats:

Object format
{
    "alphabet": string[],
    "initialStates": string[],
    "acceptingStates": string[],
    "states": string[],
    "transitions": { fromState: string, toState: string, symbols: string[] }[],
    "position"?: {
        string : {
            "x": number,
            "y": number
        }
    }
}

  • alphabet: A list of strings representing the symbols of the automaton (will be updated according to the transitions)
  • initialStates: A list of strings representing the initial states of the automaton
  • acceptingStates: A list of strings representing the final states of the automaton
  • states: A list of strings representing the states of the automaton
  • transitions: A list of objects representing the transitions
  • position: An object representing the position of the states inside the editor (the states will be placed randomly if not specified).

String format
#states
s0
s1
s2
...
#initials
s0
s1
...
#accepting
s2
#alphabet
a
b
c
...
#transitions
s0:a>s1
s1:a,c>s1
s1:b>s2

The editor will always return the object format
